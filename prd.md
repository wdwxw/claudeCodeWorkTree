## 1. 产品概述

### 1.1 背景与动机

Git Worktree 是 Git 提供的强大特性，允许开发者在同一个仓库下同时检出多个分支到不同的目录，从而并行开发、快速切换上下文。然而，原生 Git 命令行界面操作繁琐、缺乏可视化，导致大多数开发者不熟悉或回避使用 Worktree 功能。

Worktree Manager 旨在通过直观的 GUI 界面降低使用门槛，让开发者能以最少的操作快速创建、管理和切换 Git Worktree，同时保留内嵌终端的完整命令行能力。

### 1.2 目标用户

- 熟悉 Git 的中高级开发者，需要频繁切换功能分支
- 使用 Monorepo 或多模块项目的团队
- 需要并行维护多个版本（如 feature、hotfix、release）的开发者
- 希望减少 stash/checkout 操作带来的上下文切换成本的工程师

### 1.3 核心价值主张

- 一键创建 Worktree，无需记忆复杂命令
- 内嵌终端，无需切换应用即可执行命令
- 可视化管理多 Repo 及其 Worktree 列表
- 快速在外部编辑器（VS Code、iTerm2 等）中打开当前工作树
- 支持将 Worktree 代码一键合并回主仓库

## 2. 功能需求

### 2.1 功能总览

| 功能模块 | 功能描述 | 优先级 |
|----------|----------|--------|
| Repo 管理 | 添加、展示、删除本地 Git 仓库 | P0 - 核心 |
| Worktree 创建 | 基于当前分支一键创建 Worktree（随机目录名） | P0 - 核心 |
| Worktree 归档 | 移除 Worktree（执行 `git worktree remove`） | P0 - 核心 |
| 内嵌终端 | 右侧面板嵌入完整 Terminal，支持命令交互 | P0 - 核心 |
| 多 Repo 支持 | 左侧列表同时管理多个 Git 仓库 | P0 - 核心 |
| 胶囊快捷入口 | 快速在外部软件中打开当前 Worktree 路径 | P1 - 高 |
| PR / 合并 | 将当前 Worktree 变更合并至源 Repo 分支 | P1 - 高 |
| 终端日志查看 | 弹窗展示最近 500 行终端历史记录 | P2 - 中 |

### 2.2 左侧面板 — Repo & Worktree 列表

#### 2.2.1 添加 Repo

- 点击左侧面板底部「**+ 添加 Repo**」按钮
- 弹出系统原生目录选择窗口（Finder / 文件资源管理器）
- 用户选定包含 `.git` 目录的项目根目录后，系统校验该目录是否为有效 Git 仓库
- 校验通过后，左侧 Repo 列表新增该 Repo 条目（显示仓库名称）
- Repo 信息持久化存储到本地配置文件（JSON / SQLite）

#### 2.2.2 Repo 条目交互

每个 Repo 条目右侧提供两个操作入口：

**「+」（加号）按钮**

- 基于该 Repo 当前所在分支，自动创建新 Worktree
- 自动生成随机目录名，格式：`{分支名}-{随机6位字母数字}`，如 `auth-flow-a3f7k2`（分支名中的 `/` 替换为 `-`，随机部分保证全局唯一）
- Worktree 目录固定创建在父仓库同级目录下
- 创建成功后，该 Repo 条目下展开显示新的 Worktree 记录

**「...」（更多）按钮**

- 点击弹出下拉菜单，包含「删除」选项
- 点击「删除」后，**仅从软件界面移除**该 Repo 及其所有 Worktree 记录
- 实际磁盘目录及 Git 数据不做任何修改（安全操作）
- 操作需弹出确认对话框，防止误删

#### 2.2.3 Worktree 记录展示

- 每个 Repo 下方以树形/列表结构展示其所有 Worktree
- Worktree 条目显示：分支名 / 目录名 / 当前状态（活跃 / 已归档）
- 条目右侧提供「**归档**」按钮（图标样式，hover 时显示）
- 点击「归档」后，执行 `git worktree remove <path>`，移除该 Worktree
- 归档后的 Worktree 以置灰/删除线样式在列表中保留展示（或直接移除，可配置）
- 点击 Worktree 条目，右侧终端切换到该 Worktree 路径（`cd` 到对应目录）

### 2.3 右侧面板 — 内嵌终端

#### 2.3.1 终端核心功能

- 使用 xterm.js（Electron）或系统 PTY（Tauri）实现完整终端仿真
- 支持彩色输出、光标控制、键盘快捷键（Ctrl+C、Ctrl+L 等）
- 切换 Worktree 时，终端自动 `cd` 到对应 Worktree 目录
- 支持多 Tab 或 Split 终端（P2 阶段扩展）

#### 2.3.2 顶部右上角胶囊组件

胶囊为**左右双区域**设计，水平排列：

**左区域（上次使用的软件）**

- 显示上一次选择打开的外部软件名称及图标（如「VS Code」）
- 点击后以当前 Worktree 路径为项目根目录，直接启动该软件（`code .`）

**右区域（下拉软件列表）**

- 点击后展开下拉菜单，列出可用外部软件（VS Code、iTerm2、Cursor、Terminal 等）
- 用户点击某软件后：以当前 Worktree 路径打开该软件，并记录为「上次使用」
- 软件列表可在设置中自定义管理

#### 2.3.3 PR / 合并按钮

- 胶囊右侧提供「**PR**」按钮（Pull Request / 合并入口）
- 点击后弹出合并配置对话框：
  - 选择目标分支（默认为源 Repo 的主分支，如 `main` / `master`）
  - 选择合并方式：Merge / Rebase / Squash
  - 确认后在内嵌终端执行对应 git 命令，并实时展示执行结果
  - 如存在冲突，在终端中提示用户手动解决

#### 2.3.4 底部工具栏

终端下方固定显示两个功能按钮：

**「Show」按钮**

- 点击后弹出「终端日志」浮层窗口
- 显示当前终端会话最近 **500 行**历史记录
- 内容可滚动，支持全文搜索（关键词高亮）
- 右上角提供「复制全部」按钮，一键复制全部日志文本
- 浮层底部显示「关闭」按钮

**「Enter」按钮**

- 点击后在终端底部展开一个文本输入区域
- 输入命令文本后按回车键，将命令追加写入终端并执行
- 支持 Shift+Enter 换行输入多行命令
- 按 Esc 关闭输入区域

## 3. 界面设计规范

### 3.1 整体布局

采用经典**两栏布局**，最小窗口宽度 960px：

| 区域 | 尺寸 | 内容 |
|------|------|------|
| 左侧面板 | 固定宽度（240px） | Repo 列表 + Worktree 树形列表，支持折叠/展开 |
| 右侧主区域 | 弹性宽度（剩余空间） | 顶部工具栏 + 内嵌终端 + 底部操作栏 |

### 3.2 组件规范

| 组件 | 形态 | 交互说明 |
|------|------|----------|
| 胶囊组件 | 左右双区，圆角胶囊形态 | 高度 32px，背景半透明深色，hover 时高亮 |
| 归档按钮 | 图标按钮（仅图标，无文字） | 默认隐藏，鼠标 hover 到 Worktree 条目时显示 |
| + 按钮 | 小型圆形图标按钮 | 紧靠 Repo 条目名称右侧 |
| ... 按钮 | 三点图标触发下拉 | 下拉菜单包含「删除」等选项 |
| PR 按钮 | 小型文字+图标按钮 | 紧靠胶囊右侧，点击触发合并流程 |
| 终端日志浮层 | 居中模态浮层 | 宽 680px，高 480px，含搜索栏、内容区、底部操作栏 |
| 命令输入区 | 终端底部展开输入框 | 深色背景，等宽字体，与终端风格一致 |

### 3.3 视觉风格

- **主题**：深色主题（Dark Theme），与开发者常用 IDE 风格一致
- **主色调**：深灰背景 `#1E1E2E` + 蓝色强调色 `#1A56DB`
- **终端字体**：等宽字体（JetBrains Mono / SF Mono / Consolas）
- **界面字体**：系统默认 Sans-Serif（macOS: `-apple-system`，Windows: `Segoe UI`）
- **图标库**：Lucide Icons 或 Heroicons

## 4. 核心交互流程

### 4.1 添加 Repo 并创建 Worktree

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 点击「+ 添加 Repo」 | 左下角固定按钮 |
| 2 | 选择目录 | 系统文件选择对话框弹出 |
| 3 | 校验 Git 仓库 | 检测所选路径是否含 `.git` 目录 |
| 4 | Repo 出现在左侧列表 | 显示仓库名，可展开/折叠 |
| 5 | 点击 Repo 右侧「+」 | 基于当前分支创建新 Worktree |
| 6 | Worktree 创建成功 | 列表新增条目，终端 cd 到新路径 |

### 4.2 在外部软件中打开 Worktree

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 选中目标 Worktree | 左侧列表点击，终端切换到该路径 |
| 2 | 点击胶囊右区下拉 | 展开可用软件列表 |
| 3 | 选择目标软件 | 如「VS Code」 |
| 4 | 软件启动 | 以当前 Worktree 路径为工作目录打开 |
| 5 | 下次可直接点击胶囊左区 | 快速重新打开上次使用的软件 |

### 4.3 归档 Worktree

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | Hover 到 Worktree 条目 | 显示归档按钮 |
| 2 | 点击归档按钮 | 弹出确认对话框 |
| 3 | 确认归档 | 执行 `git worktree remove <path>` |
| 4 | 列表更新 | 条目从活跃列表移除（或置灰展示） |

## 5. 技术架构建议

### 5.1 技术选型

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 桌面框架 | Electron（首选）/ Tauri | Electron 生态成熟，xterm.js 集成更简单；Tauri 包体更小 |
| UI 框架 | React + TypeScript | 组件化开发，类型安全 |
| 终端组件 | xterm.js + node-pty | 业界标准方案，VS Code 同款终端库 |
| 状态管理 | Zustand / Jotai | 轻量级，适合桌面应用场景 |
| 本地存储 | electron-store / SQLite | 持久化 Repo 列表、用户偏好设置 |
| Git 操作 | simple-git（Node.js） | 封装 Git 命令，解析输出，安全执行 |
| 样式方案 | Tailwind CSS + CSS Variables | 快速开发，主题切换支持 |

### 5.2 数据模型

**Repo 实体**

```typescript
interface Repo {
  id: string           // UUID
  name: string         // 仓库名称
  path: string         // .git 目录所在绝对路径
  addedAt: number      // timestamp
  worktrees: Worktree[]
}
```

**Worktree 实体**

```typescript
interface Worktree {
  id: string                        // UUID
  repoId: string                    // 关联 Repo
  branch: string                    // 分支名
  path: string                      // Worktree 绝对路径
  status: 'active' | 'archived'
  createdAt: number                 // timestamp
}
```

### 5.3 安全性约束

- 删除 Repo 操作仅从应用数据库移除记录，**不执行任何文件系统删除**
- 归档 Worktree 执行 `git worktree remove` 前必须经用户二次确认
- PR/合并操作前展示完整 diff 摘要，用户确认后方可执行
- 终端命令输入区不对输入内容做任何过滤，保持原生命令行体验

## 6. 非功能性需求

### 6.1 性能

- 应用启动时间 < 3 秒（冷启动）
- Worktree 创建操作响应时间 < 2 秒（不含 git 执行时间）
- 终端输入响应延迟 < 50ms
- 左侧列表支持 50+ Repo、每 Repo 10+ Worktree 的流畅渲染

### 6.2 兼容性

- macOS 12+（Monterey 及以上）
- Git 版本 >= 2.17（Worktree 特性稳定版本）

### 6.3 可用性

- 首次使用提供空状态引导（Empty State），指导用户添加第一个 Repo
- 所有破坏性操作（删除、归档）须二次确认
- 操作失败时在终端或通知区显示清晰的错误信息
- 支持键盘快捷键：`Cmd+N`（新建 Worktree）、`Cmd+T`（新建终端 Tab，P2）

## 8. 决策记录

### 8.1 已确认事项

| 事项 | 决策 | 说明 |
|------|------|------|
| Worktree 默认创建路径 | 固定为父仓库同级目录，不开放自定义 | 目录名使用随机字符串保证唯一性，无需用户干预 |
| 随机目录名生成规则 | `{分支名}-{随机6位字母数字}` | 如当前分支为 `feature/auth-flow`，生成 `auth-flow-a3f7k2`；分支名中的 `/` 替换为 `-` |
| PR / 合并功能 | 仅支持本地 merge，不对接远程 API | 在内嵌终端执行 `git merge` / `git rebase`，不依赖 GitHub / GitLab |
| 收费模式 | 免费，不收费 | — |

| 目标平台 | 仅支持 macOS，不支持 Windows | 无需考虑 WSL 终端及 Windows 兼容性 |

### 8.2 技术风险

- **iTerm2 兼容性**：部分版本 iTerm2 不支持命令行参数直接打开指定目录，需要 AppleScript 调用，存在兼容性风险
- **Worktree 归档失败**：`git worktree remove` 在 Worktree 存在未提交修改时会失败，需在 UI 层提前检测并提示用户
- **大量 Worktree 性能**：多 Repo 大量 Worktree 场景下，左侧列表渲染性能需要虚拟列表优化
